About ECNOVA

ECNOVA translates execution time into an on‑chain artifact called a Time Cell. Time Cells have participants, assets (optional), rules, and a due_slot; settlement is permissionless.

Solanaslot‑based duepermissionless

About the Protocol

Time Cells

A Time Cell is a PDA account with due_slot, terms_hash and statuses (INIT→ACTIVE→DUE→SETTLED|DISPUTED|EXPIRED). Deadlines are in Solana slots; human‑readable time is UI‑only.

In ECNOVA there is no “later” — only executed or not executed.

Layers

Attention Layer (PoR‑Engine + RAG) validates human participation. App Shell is the user hub. DAO governs treasury under timelock; core programs are immutable.

DApp — App Shell

ECNOVA App Shell (Time Execution Hub): My Vesting, Create Time Cell, Dashboard, DAO panel, Buyback Tracker. Tech: React + Solana RPC + Wallet Adapter. Attention check is performed before sensitive actions.
